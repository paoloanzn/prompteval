Generate an evaluation dataset for testing prompts that triage an insurance claim
description into a structured decision.

Output format:
```json
[
  {"task": "Insurance claim narrative:\n<3-5 sentence description of incident, damage, and policy context>\n\nProduce a single triage decision in this exact format:\n  DECISION: <approve|deny|investigate|escalate>\n  REASON_CODE: <one of: covered_peril|excluded_peril|insufficient_evidence|fraud_indicator|policy_lapsed|exceeds_limit|requires_adjuster>\n  CONFIDENCE: <low|medium|high>"},
  ...
]
```

Constraints on tasks:
* Mix peril types: water damage, fire, theft, auto collision, hail, liability.
* Include policy edge cases: flood exclusion in standard homeowners, named-storm
  deductibles, vehicle use exclusion (commercial vs personal), wear-and-tear vs
  sudden damage.
* Include fraud indicators: claim filed weeks after the event, prior similar claims,
  inconsistencies between police report and claim narrative.
* Each narrative includes the policy summary inline so the answer is derivable from
  the task itself.

Generate 10 objects.
